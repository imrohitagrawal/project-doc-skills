# Simple-English rules for a grade-1 reading level

The goal: a young child could read most of it out loud and understand it. These rules are stricter
than the shared house style because this is the simplest document in the suite.

## Sentence rules
- About **8 words** per sentence. If a sentence is long, cut it into two.
- **One idea per sentence.** Do not join ideas with "and", "but", "which", or "because" if you can
  split them instead.
- Use **active voice**: "Click the button," not "The button should be clicked."
- Start steps with a **doing word**: *Open. Type. Click. Tap. Wait. Look.*

## Word rules
- Use the **common word**. Examples: use *use* (not *utilise*), *start* (not *initiate*), *open* (not
  *launch*), *check* (not *verify*), *let* (not *enable*), *get* (not *obtain*).
- If a word is hard and you must use it, **explain it right there** in tiny words: "An *app* is a
  program you open."
- No idioms or sayings (they confuse new readers and are hard to translate).

## Step rules
- **Number** every step.
- Say **exactly** what to do: what to click, type, or tap, and where it is on the screen.
- After a step, say **what they will see**, so they know it worked: "You will see a green tick."
- Keep each task **small**. If it has many steps, split it into two tasks.

## Rewrite examples (before to after)

| Too hard | Grade-1 version |
|---|---|
| "Utilise the dashboard to configure your preferences." | "Open the home screen. Pick what you like." |
| "Authentication is required prior to accessing the resource." | "First you sign in. Then you can use it." |
| "Should an error be encountered, consult the troubleshooting section." | "If you see an error, go to the help part below." |
| "Initiate the process by executing the quickstart command." | "To start, type the start command. Press Enter." |
| "The system will subsequently generate a report." | "Then it makes a report for you." |

## Pictures
- Add a simple picture for each main task. Use the project palette.
- Every picture needs **alt text** (a short sentence saying what the picture shows) and a static
  fallback.
- Show ✓ and the word "Done" for success, not colour alone.

## What to leave out
- Leave out *why* it works, the history, and the deep detail. Those belong in the learning track or
  the architecture doc. A usage guide only shows **how to do the task**.

## Read it back
After writing a section, read it slowly in your head as if you were six years old. If any sentence is
hard, cut words and split it. Then run the reading-level check; simplify until it passes the target.
